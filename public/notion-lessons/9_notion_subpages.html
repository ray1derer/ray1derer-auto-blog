Notion 완전 정복: 일상 속 문제 해결부터 시작하는 스마트 워크스페이스 구축

9강: 페이지 안에 페이지 만들기 (정보 계층화와 구조화)

강의 목표: 이번 강의에서는 Notion의 '페이지 안에 페이지' 기능을 활용하여 정보를 체계적으로 계층화하고 구조화하는 방법을 배웁니다. 복잡한 정보도 깔끔하게 정리하고, 필요한 정보를 쉽게 찾아볼 수 있도록 페이지를 구성하는 노하우를 익혀 보세요.

9.1. 왜 '페이지 안에 페이지'를 만들어야 할까요? (정보 계층화의 중요성)

Notion은 무한히 확장되는 캔버스와 같습니다. 하지만 모든 정보를 하나의 페이지에 나열하면 오히려 복잡하고 찾기 어려워집니다. '페이지 안에 페이지' 기능은 정보를 논리적인 계층으로 나누어 관리할 수 있게 해줍니다.

  • 정보의 체계적인 정리: 관련된 정보를 하나의 상위 페이지 아래에 하위 페이지로 묶어 관리할 수 있습니다. (예: '프로젝트' 페이지 아래에 '기획', '진행', '완료' 페이지)
  • 가독성 향상: 복잡한 내용을 여러 페이지로 나누어 담음으로써, 각 페이지의 내용을 간결하게 유지하고 가독성을 높일 수 있습니다.
  • 쉬운 탐색: 사이드바를 통해 페이지의 계층 구조를 한눈에 파악하고, 원하는 페이지로 빠르게 이동할 수 있습니다.

9.2. '페이지 블록'으로 하위 페이지 만들기

'페이지 블록'은 Notion에서 하위 페이지를 만드는 가장 기본적인 방법입니다.

  1. 페이지 블록 추가:
    • 새로운 하위 페이지를 만들 상위 페이지에서 /를 입력하고 페이지 또는 page를 입력하여 '페이지' 블록을 선택합니다.

    스크린샷 가이드: 슬래시 명령어로 페이지 블록 추가

    • 목적: 슬래시 명령어를 사용하여 페이지 블록을 추가하는 과정을 보여줍니다.
    • 가이드: / 입력 후 '페이지'를 검색하여 선택하는 모습을 캡처합니다.
    • 이미지 파일명: 9-1_add_page_block.png
  2. 새 페이지 생성 또는 기존 페이지 연결:
    • '새 페이지'를 선택하여 완전히 새로운 하위 페이지를 만들 수 있습니다.
    • '기존 페이지 연결'을 선택하여 이미 만들어진 페이지를 현재 페이지의 하위 페이지로 연결할 수 있습니다.

    스크린샷 가이드: 새 페이지 생성 또는 기존 페이지 연결 옵션

    • 목적: 페이지 블록 추가 후 나타나는 '새 페이지'와 '기존 페이지 연결' 옵션을 보여줍니다.
    • 가이드: 페이지 블록을 추가한 후 나타나는 팝업 메뉴를 캡처합니다.
    • 이미지 파일명: 9-2_new_or_link_page.png
  3. 하위 페이지 제목 입력:
    • 새 페이지를 생성했다면, 하위 페이지의 제목을 입력합니다.

    스크린샷 가이드: 하위 페이지 제목 입력

    • 목적: 새로 생성된 하위 페이지에 제목을 입력하는 모습을 보여줍니다.
    • 가이드: 하위 페이지 상단에 제목을 입력하는 커서와 입력된 제목이 보이는 화면을 캡처합니다.
    • 이미지 파일명: 9-3_subpage_title_input.png

9.3. 사이드바에서 페이지 구조 확인 및 관리

Notion의 왼쪽 사이드바는 여러분의 워크스페이스 전체 페이지 구조를 보여주는 내비게이션 역할을 합니다.

  1. 페이지 계층 확인:
    • 사이드바에서 페이지 이름 옆에 있는 작은 화살표(▶)를 클릭하면 하위 페이지 목록이 펼쳐집니다.

    스크린샷 가이드: 사이드바에서 페이지 계층 펼치기

    • 목적: 사이드바에서 상위 페이지 아래에 하위 페이지가 계층적으로 표시되는 모습을 보여줍니다.
    • 가이드: 사이드바에서 상위 페이지 이름 옆의 화살표를 클릭하여 하위 페이지 목록이 펼쳐진 화면을 캡처합니다.
    • 이미지 파일명: 9-4_sidebar_page_hierarchy.png
  2. 페이지 이동 및 재배열:
    • 사이드바에서 페이지를 드래그 앤 드롭하여 다른 페이지의 하위 페이지로 만들거나, 페이지의 순서를 변경할 수 있습니다.

    스크린샷 가이드: 사이드바에서 페이지 이동 및 재배열

    • 목적: 사이드바에서 페이지를 드래그하여 다른 페이지의 하위 페이지로 만드는 모습을 보여줍니다.
    • 가이드: 사이드바에서 페이지를 드래그했을 때 파란색 가이드라인이 나타나 페이지가 삽입될 위치를 알려주는 화면을 캡처합니다.
    • 이미지 파일명: 9-5_sidebar_drag_reorder.png

9.4. 페이지 링크와 페이지 임베드 (정보 연결의 다양한 방법)

'페이지 블록' 외에도 페이지를 연결하고 통합하는 다양한 방법이 있습니다.

  1. 페이지 링크 (Link to page):
    • 특정 페이지의 내용을 현재 페이지에 직접 표시하지 않고, 해당 페이지로 이동할 수 있는 링크를 만듭니다.
    • /를 입력하고 페이지 링크 또는 link to page를 입력하여 블록을 선택한 후, 연결하고 싶은 페이지를 검색하여 선택합니다.

    스크린샷 가이드: 페이지 링크 블록 추가 및 결과

    • 목적: 페이지 링크 블록이 페이지에 어떻게 표시되는지 보여줍니다.
    • 가이드: 페이지 링크 블록을 추가한 후, 링크된 페이지의 제목이 표시된 화면을 캡처합니다.
    • 이미지 파일명: 9-6_link_to_page_block.png
  2. 페이지 임베드 (Embed page):
    • 다른 Notion 페이지의 내용을 현재 페이지에 직접 삽입하여 보여줍니다. 원본 페이지의 내용이 변경되면 임베드된 내용도 실시간으로 업데이트됩니다.
    • /를 입력하고 페이지 임베드 또는 embed page를 입력하여 블록을 선택한 후, 임베드하고 싶은 페이지를 검색하여 선택합니다.

    스크린샷 가이드: 페이지 임베드 블록 추가 및 결과

    • 목적: 페이지 임베드 블록이 페이지에 어떻게 표시되는지 보여줍니다.
    • 가이드: 페이지 임베드 블록을 추가한 후, 임베드된 페이지의 내용이 현재 페이지에 직접 표시된 화면을 캡처합니다.
    • 이미지 파일명: 9-7_embed_page_block.png

9.5. 정보 구조화 예시: 프로젝트 관리 워크스페이스

'페이지 안에 페이지' 기능을 활용하여 프로젝트 관리 워크스페이스를 어떻게 구조화할 수 있는지 간단한 예시를 통해 살펴보겠습니다.

  • 상위 페이지: '프로젝트 관리'
    • 하위 페이지: '진행 중인 프로젝트'
      • 하위 페이지: 'Notion 강의 제작'
        • 하위 페이지: '1강 - Notion 왜 필요할까요?'
        • 하위 페이지: '2강 - Notion 설치 및 워크스페이스'
        • 하위 페이지: '3강 - Notion으로 메모장 만들기'
    • 하위 페이지: '완료된 프로젝트'
    • 하위 페이지: '아이디어 보관함'

이처럼 페이지를 계층적으로 구성하면, 복잡한 프로젝트도 깔끔하게 정리하고 필요한 정보에 빠르게 접근할 수 있습니다.

강의 요약:

이번 9강에서는 Notion의 '페이지 안에 페이지' 기능을 통해 정보를 체계적으로 계층화하고 구조화하는 방법을 배웠습니다. '페이지 블록'을 사용하여 하위 페이지를 만들고, 사이드바를 통해 페이지 구조를 관리하며, '페이지 링크'와 '페이지 임베드'로 정보를 연결하는 다양한 방법을 익혔습니다. 이제 여러분은 Notion에서 복잡한 정보도 효율적으로 정리할 수 있습니다.
