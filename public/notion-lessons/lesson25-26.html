25강 & 26강: 최종 프로젝트 1 - PARA 메소드로 '세컨드 브레인' 구축하기 (2부작)

이 최종 프로젝트는 노션의 모든 기능을 통합하여 개인의 삶과 업무를 총체적으로 관리하는 '라이프 운영체제(Life Operating System)'를 구축하는 데 중점을 둡니다. PARA 메소드는 정보와 작업을 체계적으로 정리하고 관리하는 강력한 프레임워크입니다. 이 2부작 강의에서는 PARA 메소드의 원칙을 이해하고, 이를 노션에서 구현하여 자신만의 '세컨드 브레인'을 구축하는 방법을 배웁니다.

학습 목표

  • 프로젝트(Projects), 영역(Areas), 자료(Resources), 보관(Archives)의 원칙을 이해합니다.
  • 완전하고 상호 연결된 PARA 시스템을 처음부터 구축합니다.

주요 개념

  • PARA 메소드: Tiago Forte가 제안한 정보 정리 및 관리 시스템으로, 모든 디지털 정보를 네 가지 최상위 카테고리(Projects, Areas, Resources, Archives)로 분류합니다.
  • 프로젝트 (Projects): 특정 목표를 달성하기 위한 일련의 작업으로, 시작과 끝이 명확합니다. (예: '새 웹사이트 런칭', '노션 강좌 제작')
  • 영역 (Areas): 지속적으로 관리해야 하는 책임 영역으로, 끝이 없습니다. (예: '건강', '재정', '개인 성장')
  • 자료 (Resources): 관심 있는 주제나 유용한 정보의 저장소입니다. (예: '노션 팁', '마케팅 자료')
  • 보관 (Archives): 더 이상 활성화되지 않지만 나중에 참조할 수 있는 모든 항목입니다.
  • 실행 가능성 (Actionability): 정보가 얼마나 즉각적으로 행동으로 이어질 수 있는지를 나타냅니다. PARA는 실행 가능성을 기준으로 정보를 분류합니다.
  • 정보 아카이빙 (Information Archiving): 완료된 프로젝트나 더 이상 필요하지 않은 자료를 체계적으로 보관하여 워크스페이스를 깔끔하게 유지하고 필요할 때 쉽게 찾아볼 수 있도록 합니다.

실습 과제 (2강에 걸쳐 진행)

1부: PARA 핵심 데이터베이스 설정

  • 4개의 핵심 데이터베이스('프로젝트', '영역', '자료', '보관')를 생성합니다.
  • 이들 데이터베이스 간의 중요한 관계(예: '프로젝트'와 '영역' 연결, '자료'와 '영역' 연결 등)를 설정하여 상호 연결된 시스템을 만듭니다.

2부: PARA 메인 대시보드 및 자동화 구현

  • 각 카테고리(프로젝트, 영역, 자료)에 대한 연결된 보기가 있는 메인 대시보드를 구축합니다.
  • 관련 업무와 자료를 자동으로 필터링하는 프로젝트 및 영역 템플릿을 만듭니다.
  • '보관' 체크박스 자동화를 구현합니다. (예: 프로젝트가 완료되면 자동으로 '보관' 데이터베이스로 이동하거나, '보관' 속성이 체크되도록 설정)
