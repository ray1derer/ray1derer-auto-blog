Notion 완전 정복: 일상 속 문제 해결부터 시작하는 스마트 워크스페이스 구축

4강: Notion으로 '캘린더' 만들기 (일정 관리의 시작)

강의 목표: 이번 강의에서는 Notion의 강력한 기능 중 하나인 '데이터베이스'를 활용하여 캘린더를 만들고 일정을 관리하는 방법을 배웁니다. Notion 캘린더의 유연성과 편리함을 경험해 보세요.

4.1. Notion 데이터베이스의 '캘린더 뷰' 소개

Notion의 캘린더는 단순히 날짜를 표시하는 것을 넘어, 여러분의 모든 정보를 일정과 연결하여 관리할 수 있는 강력한 도구입니다. Notion의 캘린더는 '데이터베이스'의 한 종류인 '캘린더 뷰'를 통해 구현됩니다.

  • 데이터베이스: Notion에서 정보를 체계적으로 관리하는 가장 강력한 도구입니다. 스프레드시트와 비슷하지만, 각 행(아이템)이 하나의 독립적인 페이지가 될 수 있고, 다양한 방식으로 정보를 볼 수 있는 '뷰'를 제공합니다.
  • 캘린더 뷰: 데이터베이스에 날짜 속성이 포함되어 있다면, 그 데이터를 캘린더 형태로 볼 수 있게 해주는 기능입니다.

4.2. 새로운 캘린더 데이터베이스 생성하기

캘린더를 만드는 방법은 두 가지가 있습니다: '인라인 캘린더'와 '전체 페이지 캘린더'.

  1. 새 페이지 생성 또는 기존 페이지에 추가:
    • 새로운 캘린더를 위한 페이지를 만들거나, 기존 페이지에 캘린더를 추가할 위치로 이동합니다.

    스크린샷 가이드: 새 페이지 생성 또는 기존 페이지에 커서 위치

    • 목적: 캘린더를 추가할 준비가 된 Notion 페이지 화면을 보여줍니다.
    • 가이드: Notion 워크스페이스에서 새로운 빈 페이지를 열거나, 기존 페이지의 내용 아래에 커서가 깜빡이는 화면을 캡처합니다.
    • 이미지 파일명: 4-1_prepare_for_calendar.png
  2. 캘린더 데이터베이스 블록 추가:
    • 새로운 블록을 만들 위치에서 /를 입력하고 캘린더 또는 calendar를 입력하여 '캘린더' 블록을 선택합니다.

    스크린샷 가이드: 슬래시 명령어로 캘린더 블록 추가

    • 목적: 슬래시 명령어를 사용하여 캘린더 블록을 추가하는 과정을 보여줍니다.
    • 가이드: / 입력 후 '캘린더'를 검색하여 선택하는 모습을 캡처합니다.
    • 이미지 파일명: 4-2_add_calendar_block.png
  3. 새 데이터베이스 생성 선택:
    • '새 데이터베이스 생성' 옵션을 선택합니다. (기존 데이터베이스를 캘린더 뷰로 볼 수도 있지만, 여기서는 새로 만듭니다.)

    스크린샷 가이드: 새 데이터베이스 생성 선택

    • 목적: 캘린더 블록 추가 후 나타나는 옵션에서 '새 데이터베이스 생성'을 선택하는 모습을 보여줍니다.
    • 가이드: '새 데이터베이스 생성' 버튼이 강조된 화면을 캡처합니다.
    • 이미지 파일명: 4-3_select_new_db.png
  4. 캘린더 이름 지정:
    • 생성된 캘린더의 제목을 입력합니다. (예: "나의 일정 캘린더")

    스크린샷 가이드: 캘린더 이름 지정

    • 목적: 새로 생성된 캘린더의 제목을 입력하는 모습을 보여줍니다.
    • 가이드: 캘린더 상단에 제목을 입력하는 커서와 입력된 제목이 보이는 화면을 캡처합니다.
    • 이미지 파일명: 4-4_calendar_name_input.png

4.3. 일정 추가 및 속성 설정하기

캘린더에 일정을 추가하고, 각 일정에 필요한 정보를 '속성(Property)'으로 설정할 수 있습니다.

  1. 새 일정 추가:
    • 캘린더에서 원하는 날짜를 클릭하거나, 캘린더 우측 상단의 '새로 만들기' 버튼을 클릭합니다.

    스크린샷 가이드: 캘린더에 새 일정 추가

    • 목적: 캘린더에서 날짜를 클릭하여 새 일정을 추가하는 모습을 보여줍니다.
    • 가이드: 캘린더에서 특정 날짜를 클릭했을 때 나타나는 새 일정 팝업 창을 캡처합니다.
    • 이미지 파일명: 4-5_add_new_event.png
  2. 일정 제목 입력:
    • 새 일정 페이지가 열리면, 일정의 제목을 입력합니다. (예: "친구와 저녁 식사")

    스크린샷 가이드: 일정 제목 입력

    • 목적: 새 일정 페이지에서 제목을 입력하는 모습을 보여줍니다.
    • 가이드: 일정 페이지 상단에 제목을 입력하는 커서와 입력된 제목이 보이는 화면을 캡처합니다.
    • 이미지 파일명: 4-6_event_title_input.png
  3. 날짜 속성 설정:
    • 기본적으로 '날짜' 속성이 자동으로 추가되어 있습니다. 클릭하여 시작일과 종료일을 설정할 수 있습니다. '종료일 포함'을 활성화하면 여러 날짜에 걸친 일정을 만들 수 있습니다.

    스크린샷 가이드: 날짜 속성 설정

    • 목적: 일정 페이지에서 날짜 속성을 설정하는 모습을 보여줍니다.
    • 가이드: '날짜' 속성을 클릭했을 때 나타나는 달력 팝업과 시작일/종료일 설정 옵션을 캡처합니다.
    • 이미지 파일명: 4-7_set_date_property.png
  4. 시간 속성 설정 (선택 사항):
    • 날짜 속성에서 '시간 포함'을 활성화하면 특정 시간을 지정할 수 있습니다.

    스크린샷 가이드: 시간 속성 설정

    • 목적: 일정 페이지에서 시간 속성을 설정하는 모습을 보여줍니다.
    • 가이드: 날짜 속성 설정 팝업에서 '시간 포함'을 활성화하고 시간을 설정하는 모습을 캡처합니다.
    • 이미지 파일명: 4-8_set_time_property.png
  5. 새로운 속성 추가하기 (태그, 상태 등):
    • 일정 페이지에서 '속성 추가' 버튼을 클릭하여 필요한 정보를 추가할 수 있습니다.
    • '선택(Select)' 속성: 미리 정의된 옵션 중 하나를 선택합니다. (예: '카테고리' - 개인, 업무, 학습)
    • '다중 선택(Multi-select)' 속성: 여러 옵션을 동시에 선택합니다. (예: '참석자' - 여러 사람 이름)
    • '상태(Status)' 속성: 할 일의 진행 상태를 나타냅니다. (예: '진행 전', '진행 중', '완료')

    스크린샷 가이드: 속성 추가 및 유형 선택

    • 목적: 일정 페이지에서 새로운 속성을 추가하고 속성 유형을 선택하는 모습을 보여줍니다.
    • 가이드: '속성 추가' 버튼을 클릭했을 때 나타나는 속성 유형 목록을 캡처합니다.
    • 이미지 파일명: 4-9_add_new_property.png
  6. 일정 페이지 내용 추가:
    • 일정 페이지는 일반 Notion 페이지와 동일하게 텍스트, 이미지, 체크리스트 등 다양한 블록을 추가하여 상세 내용을 기록할 수 있습니다.

    예제:

    "친구와 저녁 식사" 일정 페이지에 약속 장소, 메뉴 아이디어, 준비물 체크리스트 추가

    스크린샷 가이드: 일정 페이지에 내용 추가

    • 목적: 일정 페이지에 텍스트나 다른 블록을 추가하는 모습을 보여줍니다.
    • 가이드: 일정 제목 아래에 텍스트를 입력하거나 블록을 추가하는 모습을 캡처합니다.
    • 이미지 파일명: 4-10_add_content_to_event_page.png

강의 요약:

이번 4강에서는 Notion의 '데이터베이스' 기능을 활용하여 캘린더를 만들고 일정을 관리하는 방법을 배웠습니다. 캘린더 뷰를 통해 일정을 시각적으로 확인하고, 다양한 속성을 추가하여 일정에 대한 상세 정보를 체계적으로 기록할 수 있음을 확인했습니다.
