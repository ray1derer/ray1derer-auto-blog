Notion 완전 정복: 일상 속 문제 해결부터 시작하는 스마트 워크스페이스 구축

11강: 동기화 블록과 연결된 페이지 (정보의 재활용과 효율성)

강의 목표: 이번 강의에서는 Notion의 '동기화 블록'과 '연결된 페이지' 기능을 활용하여 정보를 효율적으로 재활용하고, 여러 페이지에서 동일한 내용을 실시간으로 공유하는 방법을 배웁니다. 정보 중복을 줄이고 워크스페이스의 효율성을 극대화해 보세요.

11.1. 동기화 블록 (Synced Block): 여러 페이지에서 동일한 내용 공유하기

'동기화 블록'은 Notion의 강력한 기능 중 하나로, 하나의 블록을 여러 페이지에 복사하여 붙여넣었을 때, 원본 블록의 내용이 변경되면 복사된 모든 블록의 내용도 실시간으로 함께 변경되는 기능입니다.

  • 정보의 일관성 유지: 중요한 공지사항, 자주 묻는 질문(FAQ), 핵심 가이드라인 등 여러 곳에 동일하게 표시되어야 하는 정보를 관리할 때 매우 유용합니다.
  • 업데이트의 용이성: 한 곳에서만 내용을 수정하면 모든 관련 페이지에 자동으로 반영되므로, 정보 업데이트에 드는 시간과 노력을 크게 줄일 수 있습니다.
  1. 동기화 블록 생성:
    • 동기화하고 싶은 블록(텍스트, 이미지, 체크리스트 등 어떤 블록이든 가능)을 선택합니다.
    • 블록 메뉴(⋮⋮ 아이콘 클릭)에서 '동기화 블록으로 전환' 또는 'Turn into Synced block'을 선택합니다.

    스크린샷 가이드: 블록을 동기화 블록으로 전환

    • 목적: 기존 블록을 동기화 블록으로 만드는 과정을 보여줍니다.
    • 가이드: 블록 메뉴에서 '동기화 블록으로 전환' 옵션을 클릭하는 모습을 캡처합니다.
    • 이미지 파일명: 11-1_turn_into_synced_block.png
  2. 동기화 블록 복사 및 붙여넣기:
    • 생성된 동기화 블록 위에 마우스를 올리면 나타나는 '복사' 버튼을 클릭합니다.
    • 동기화 블록을 삽입하고 싶은 다른 페이지로 이동하여 붙여넣기(Ctrl+V 또는 Cmd+V)를 합니다.

    스크린샷 가이드: 동기화 블록 복사 및 다른 페이지에 붙여넣기

    • 목적: 동기화 블록을 복사하여 다른 페이지에 붙여넣는 과정을 보여줍니다.
    • 가이드: 동기화 블록의 '복사' 버튼을 클릭한 후, 다른 페이지에 붙여넣었을 때 동기화 블록이 나타난 화면을 캡처합니다.
    • 이미지 파일명: 11-2_copy_paste_synced_block.png
  3. 동기화 확인:
    • 원본 동기화 블록의 내용을 수정하면, 복사된 모든 동기화 블록의 내용도 실시간으로 변경되는 것을 확인할 수 있습니다.

    스크린샷 가이드: 동기화 블록 내용 변경 확인

    • 목적: 원본 동기화 블록의 내용이 변경되었을 때, 복사된 블록의 내용도 함께 변경되는 모습을 보여줍니다.
    • 가이드: 원본 동기화 블록의 내용을 수정한 후, 다른 페이지에 있는 동기화 블록의 내용이 자동으로 업데이트된 화면을 캡처합니다.
    • 이미지 파일명: 11-3_synced_block_update.png

11.2. 연결된 페이지 (Link to Page): 기존 페이지를 다른 페이지에 삽입하기

'연결된 페이지(Link to Page)' 블록은 기존에 만들어둔 Notion 페이지를 현재 페이지에 삽입하는 기능입니다. 이는 '페이지 안에 페이지' 기능과 유사하지만, 기존 페이지를 재활용한다는 점에서 차이가 있습니다.

  • 정보 중복 방지: 이미 존재하는 페이지의 내용을 다시 만들 필요 없이, 해당 페이지를 현재 페이지에 연결하여 보여줄 수 있습니다.
  • 중앙 집중식 관리: 원본 페이지의 내용만 관리하면 되므로, 여러 곳에 흩어진 정보의 일관성을 유지하기 쉽습니다.
  1. 연결된 페이지 블록 추가:
    • 새로운 블록을 만들 위치에서 /를 입력하고 페이지 연결 또는 link to page를 입력하여 '페이지 연결' 블록을 선택합니다.

    스크린샷 가이드: 슬래시 명령어로 페이지 연결 블록 추가

    • 목적: 슬래시 명령어를 사용하여 페이지 연결 블록을 추가하는 과정을 보여줍니다.
    • 가이드: / 입력 후 '페이지 연결'을 검색하여 선택하는 모습을 캡처합니다.
    • 이미지 파일명: 11-4_add_link_to_page_block.png
  2. 연결할 페이지 선택:
    • 연결하고 싶은 기존 Notion 페이지를 검색하여 선택합니다.

    스크린샷 가이드: 연결할 페이지 선택

    • 목적: 페이지 연결 블록 추가 후 나타나는 검색창에서 연결할 페이지를 선택하는 모습을 보여줍니다.
    • 가이드: 검색창에 페이지 이름을 입력하여 검색 결과가 나타나고, 그 중 하나를 선택하는 모습을 캡처합니다.
    • 이미지 파일명: 11-5_select_page_to_link.png
  3. 연결된 페이지 확인:
    • 선택한 페이지가 현재 페이지에 링크 형태로 삽입됩니다. 이 링크를 클릭하면 원본 페이지로 이동합니다.

    스크린샷 가이드: 연결된 페이지 블록 결과

    • 목적: 연결된 페이지 블록이 페이지에 어떻게 표시되는지 보여줍니다.
    • 가이드: 연결된 페이지 블록이 현재 페이지에 삽입된 모습을 캡처합니다.
    • 이미지 파일명: 11-6_linked_page_result.png

11.3. 동기화 블록과 연결된 페이지의 활용 시나리오

  • 팀 공지사항: '주요 공지사항' 동기화 블록을 팀 워크스페이스의 모든 팀 페이지에 삽입합니다. 공지 내용이 변경되면 한 곳에서만 수정해도 모든 페이지에 자동으로 반영됩니다.
  • 자주 묻는 질문(FAQ): 'FAQ' 페이지를 만들고, 이 페이지를 여러 제품 설명 페이지나 고객 지원 페이지에 '연결된 페이지'로 삽입합니다. FAQ 내용이 업데이트되면 원본 페이지에서만 수정하면 됩니다.
  • 개인 대시보드: '오늘의 할 일' 동기화 블록을 개인 대시보드와 주간 계획 페이지에 모두 삽입하여, 어디서든 할 일을 확인하고 관리할 수 있도록 합니다.
  • 프로젝트 요약: '프로젝트 요약' 페이지를 만들고, 이 페이지를 '프로젝트 관리' 데이터베이스의 각 프로젝트 페이지에 '연결된 페이지'로 삽입하여, 각 프로젝트의 개요를 빠르게 확인할 수 있도록 합니다.

11.4. 실습 예제: '팀 공용 푸터' 만들고 여러 페이지에 동기화하기

이론을 배웠으니 직접 동기화 블록을 활용해 볼까요? 여러 프로젝트 페이지 하단에 공통으로 들어갈 '팀 공용 푸터'를 만들어 보겠습니다.

  1. 푸터 내용 작성: 먼저 아무 페이지에나 푸터로 사용할 내용을 블록으로 만듭니다. (예: 팀 노션 가이드 링크, 비상 연락망 등)
  2. 동기화 블록으로 전환: 작성한 블록들을 모두 선택한 후, 블록 메뉴(⋮⋮)에서 '동기화 블록으로 전환'을 클릭합니다. 블록들 주변에 빨간색 테두리가 생깁니다.
  3. 동기화 블록 복사: 생성된 동기화 블록의 메뉴에서 '복사'를 클릭합니다.
  4. 다른 페이지에 붙여넣기: 이제 이 푸터를 넣고 싶은 다른 프로젝트 페이지(예: 'A 프로젝트', 'B 프로젝트' 페이지)로 이동하여 맨 아래에 붙여넣기(Ctrl+V 또는 Cmd+V)를 합니다.
  5. 내용 수정 및 확인: 아무 페이지에서나 푸터의 내용을 수정해 보세요. 다른 페이지에 있는 푸터 내용도 실시간으로 함께 변경되는 것을 확인할 수 있습니다.

스크린샷 가이드: 여러 페이지에 동기화된 푸터

  • 목적: 하나의 동기화 블록이 여러 페이지에 동일하게 적용된 모습을 보여줍니다.
  • 가이드: 'A 프로젝트' 페이지와 'B 프로젝트' 페이지 하단에 동일한 동기화 블록(빨간색 테두리)이

    강의 요약:

    이번 11강에서는 Notion의 '동기화 블록'과 '연결된 페이지' 기능을 통해 정보를 효율적으로 재활용하고, 여러 페이지에서 동일한 내용을 실시간으로 공유하는 방법을 배웠습니다. 이 기능들을 활용하면 정보 중복을 줄이고, 워크스페이스의 일관성을 유지하며, 전반적인 효율성을 크게 높일 수 있습니다.
