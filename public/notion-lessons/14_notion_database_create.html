14강: 첫 데이터베이스 만들기 (표, 보드, 캘린더 뷰)

강의 목표: 이번 강의에서는 Notion 데이터베이스를 직접 만들어보고, 가장 기본적이면서도 강력한 3가지 보기(View)인 표, 보드, 캘린더를 활용하는 방법을 배웁니다. 이론으로만 알던 데이터베이스를 실제 워크스페이스에 적용해 보세요.

14.1. 새로운 데이터베이스 생성하기 (인라인 vs 전체 페이지)

데이터베이스를 만드는 방법은 간단합니다. 페이지에서 /데이터베이스를 입력하면 두 가지 옵션이 나타납니다.

  • 인라인(Inline): 현재 페이지 안에 표 형태로 삽입됩니다. 데이터베이스 위아래로 다른 텍스트나 이미지 블록을 추가할 수 있어 유연합니다.
  • 전체 페이지(Full page): 데이터베이스 자체가 하나의 독립된 페이지가 됩니다. 사이드바에 별도로 표시되며, 오직 데이터베이스 정보에만 집중할 수 있습니다.

스크린샷 가이드: 데이터베이스 생성 옵션

  • 목적: 슬래시 명령어로 데이터베이스를 생성할 때 나타나는 '인라인'과 '전체 페이지' 옵션을 보여줍니다.
  • 가이드: 페이지에 /데이터베이스를 입력했을 때 나타나는 팝업 메뉴를 캡처합니다.
  • 이미지 파일명: 14-1_create_database_options.png

14.2. 표(Table) 뷰: 기본 속성 추가 및 데이터 입력

데이터베이스를 처음 만들면 기본적으로 표(Table) 뷰가 생성됩니다. 표 뷰는 스프레드시트와 가장 유사한 형태로, 데이터를 입력하고 속성을 관리하기에 가장 편리합니다.

  1. 항목(페이지) 추가: 표의 맨 아래 '새로 만들기' 버튼을 클릭하여 새로운 행(데이터 항목)을 추가합니다. 각 행은 하나의 독립된 페이지입니다.
  2. 속성(열) 추가: 표의 오른쪽 끝에 있는 + 버튼을 클릭하여 새로운 속성(열)을 추가합니다. '텍스트', '숫자', '날짜', '상태' 등 다양한 유형의 속성을 선택할 수 있습니다.

스크린샷 가이드: 표 뷰에서 속성 추가 및 데이터 입력

  • 목적: 표 뷰의 기본 구조와 속성(열) 및 항목(행)을 추가하는 모습을 보여줍니다.
  • 가이드: '독서 기록장' 예시로 '책 이름', '상태', '장르', '평점' 속성이 추가된 표 뷰를 캡처합니다. 몇 개의 데이터가 입력된 상태면 좋습니다.
  • 이미지 파일명: 14-2_table_view_with_properties.png

14.3. 보드(Board) 뷰: 칸반 보드로 상태 관리하기

보드(Board) 뷰는 데이터를 특정 속성을 기준으로 그룹화하여 보여주는 칸반(Kanban) 스타일의 보기 방식입니다. 주로 프로젝트의 진행 상태나 작업 단계를 시각적으로 관리할 때 유용합니다.

  1. 보드 뷰 추가: 데이터베이스 이름 옆의 + 버튼을 클릭하고 '보드'를 선택합니다.
  2. 그룹화 기준 설정: '그룹화 기준' 옵션에서 '상태'나 '선택', '다중 선택'과 같은 속성을 선택합니다.
  3. 카드 이동: 생성된 보드에서 카드를 드래그 앤 드롭하여 다른 열로 옮기면, 해당 카드의 속성값이 자동으로 변경됩니다. (예: '할 일' 열의 카드를 '진행 중' 열로 옮기기)

스크린샷 가이드: 상태별로 그룹화된 보드 뷰

  • 목적: 동일한 데이터를 보드 뷰로 전환하고, '상태' 속성을 기준으로 그룹화한 모습을 보여줍니다.
  • 가이드: '독서 기록장' 데이터베이스를 '상태' 속성('읽을 책', '읽는 중', '완독')을 기준으로 그룹화한 보드 뷰를 캡처합니다.
  • 이미지 파일명: 14-3_board_view_by_status.png

14.4. 캘린더(Calendar) 뷰: 날짜 속성을 기준으로 일정 시각화하기

캘린더(Calendar) 뷰는 '날짜' 속성이 있는 데이터를 달력 형태로 보여줍니다. 마감일, 이벤트, 발행일 등 날짜 기반의 일정을 관리하는 데 최적화되어 있습니다.

  1. 캘린더 뷰 추가: + 버튼을 클릭하고 '캘린더'를 선택합니다.
  2. 날짜 속성 연결: Notion이 데이터베이스에 있는 '날짜' 속성을 자동으로 인식하여 캘린더에 표시합니다. 만약 날짜 속성이 여러 개라면 어떤 것을 기준으로 표시할지 선택할 수 있습니다.

스크린샷 가이드: 날짜 기준으로 표시된 캘린더 뷰

  • 목적: '날짜' 속성을 가진 데이터를 캘린더 뷰로 시각화한 모습을 보여줍니다.
  • 가이드: '독서 기록장'에 '완독일'이라는 날짜 속성을 추가하고, 해당 날짜에 책 제목이 표시된 캘린더 뷰를 캡처합니다.
  • 이미지 파일명: 14-4_calendar_view_by_date.png

14.5. 실습 예제: 나만의 '독서 기록장' 만들기

지금까지 배운 내용을 종합하여 '독서 기록장' 데이터베이스를 직접 만들어 봅시다.

  1. 데이터베이스 생성: /데이터베이스 명령어로 '독서 기록장'이라는 이름의 전체 페이지 데이터베이스를 만듭니다.
  2. 속성 설정 (표 뷰):
    • 기존 '태그' 속성의 이름을 '장르'로 바꾸고 유형을 '다중 선택'으로 변경합니다.
    • + 버튼을 눌러 '상태'(상태 유형), '저자'(텍스트 유형), '평점'(선택 유형, ⭐~⭐⭐⭐⭐⭐), '완독일'(날짜 유형) 속성을 추가합니다.
  3. 데이터 입력: 읽었거나 읽고 싶은 책 3~4권을 표에 입력합니다.
  4. 보드 뷰 생성: + 보기 추가를 눌러 '보드' 뷰를 만들고, 그룹화 기준을 '상태'로 설정합니다.
  5. 캘린더 뷰 생성: + 보기 추가를 눌러 '캘린더' 뷰를 만듭니다. '완독일'을 입력한 책들이 달력에 표시되는지 확인합니다.

스크린샷 가이드: 완성된 독서 기록장 (다양한 뷰)

  • 목적: 하나의 데이터베이스가 여러 뷰(표, 보드, 캘린더)를 가질 수 있음을 종합적으로 보여줍니다.
  • 가이드: 완성된 '독서 기록장' 데이터베이스의 탭에 '표', '보드', '캘린더' 뷰가 모두 추가된 화면을 캡처합니다.
  • 이미지 파일명: 14-5_reading_tracker_example.png

강의 요약:

이번 14강에서는 Notion 데이터베이스를 직접 생성하고, 가장 기본적인 세 가지 뷰(View)인 표, 보드, 캘린더를 설정하는 방법을 배웠습니다. 중요한 점은 이 모든 뷰가 동일한 원본 데이터를 다른 방식으로 보여주는 것이라는 사실입니다. 이제 여러분은 관리하고 싶은 정보의 성격에 따라 최적의 형태로 데이터를 시각화하고 관리할 수 있게 되었습니다.
