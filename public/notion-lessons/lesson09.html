9강: 결정적 전환 - 데이터베이스 입문

노션의 진정한 힘은 '데이터베이스'에서 나옵니다. 이 강의는 노션 사용의 패러다임을 '워드 프로세서'에서 '데이터 관리자'로 전환하는 중요한 전환점입니다. 비구조화된 페이지와 구조화된 데이터베이스의 근본적인 차이를 이해하고, 데이터베이스의 핵심 구성 요소를 파악하여 노션의 무한한 가능성을 열어봅니다.

학습 목표

  • 비구조화된 페이지와 구조화된 데이터베이스의 근본적인 차이를 이해합니다.
  • 인라인 및 전체 페이지 데이터베이스를 모두 생성할 수 있습니다.
  • 핵심 구성 요소인 항목(페이지), 속성(열), 보기를 파악합니다.

주요 개념

  • 데이터베이스 (Database): 노션에서 정보를 구조화하고 관리하는 강력한 도구입니다. 스프레드시트와 유사하지만, 각 행(항목)이 그 자체로 완전한 노션 페이지라는 점에서 훨씬 유연합니다.
  • 인라인 데이터베이스 (Inline Database) vs. 전체 페이지 데이터베이스 (Full-page Database):
    • 인라인: 기존 페이지 내에 삽입되어 페이지 콘텐츠의 일부로 존재합니다.
    • 전체 페이지: 독립적인 페이지로 존재하며, 해당 페이지 전체가 데이터베이스로 구성됩니다.
  • 항목 (Item) / 페이지 (Page): 데이터베이스의 각 행은 하나의 '항목'이며, 동시에 하나의 완전한 노션 '페이지'입니다. 이 페이지 안에는 다른 블록이나 하위 페이지를 자유롭게 추가할 수 있습니다.
  • 속성 (Property) / 열 (Column): 데이터베이스의 각 열은 '속성'이라고 불립니다. 텍스트, 숫자, 날짜, 체크박스 등 다양한 유형의 속성을 통해 항목에 대한 정보를 저장하고 분류합니다.
  • 보기 (View): 동일한 데이터베이스의 데이터를 다양한 방식으로 시각화하여 보여주는 기능입니다. 표, 보드, 갤러리, 캘린더 등 여러 보기를 통해 데이터를 목적에 맞게 활용할 수 있습니다.

전문가 팁

이 강의는 전체 과정에서 가장 중요한 전환점입니다. 노션의 사용 패러다임이 '워드 프로세서'에서 '데이터 관리자'로 넘어가는 단계임을 강조합니다. 특히 데이터베이스의 모든 행(항목)이 그 자체로 하나의 완전한 노션 페이지라는 점을 명확히 설명하는 것이 중요합니다. 이 개념을 이해하면 노션의 활용 범위가 무한히 확장됩니다.

실습 과제

'나의 영화 감상 목록'이라는 새 페이지를 만듭니다. 페이지 내에 인라인 표 데이터베이스를 생성하고, '영화 제목'(제목 속성), '장르'(텍스트 속성), '시청 여부'(체크박스 속성)라는 3개의 열을 만듭니다. 영화 3개를 추가하고, 그중 하나를 클릭하여 열리는 페이지를 확인합니다.
