10강: 데이터베이스 보기 둘러보기 - 데이터 시각화

노션 데이터베이스의 강력함은 동일한 데이터를 다양한 방식으로 시각화하여 보여주는 '보기(View)' 기능에서 나옵니다. 이 강의에서는 6가지 기본 데이터베이스 보기를 살펴보고, 각 보기가 어떤 상황에서 유용하게 사용될 수 있는지 이해하며, 데이터를 효과적으로 시각화하는 방법을 배웁니다.

학습 목표

  • 6가지 기본 데이터베이스 보기를 추가하고 설정합니다.
  • 각 보기의 주요 사용 사례를 이해합니다.

주요 개념

  • 표 보기 (Table View): 데이터를 행과 열로 구성된 스프레드시트 형태로 보여줍니다. 가장 기본적인 보기이며, 데이터를 빠르게 입력하고 편집하는 데 용이합니다.
  • 보드 보기 (Board View / Kanban): 데이터를 칸반 보드 형태로 보여줍니다. 주로 '상태' 속성을 기준으로 그룹화하여 프로젝트 진행 상황이나 워크플로우를 시각적으로 파악하는 데 사용됩니다.
  • 갤러리 보기 (Gallery View): 각 항목을 카드 형태로 보여주며, 이미지 속성이 있는 경우 이미지를 크게 표시하여 시각적인 정보를 강조합니다. 포트폴리오, 레시피, 제품 목록 등에 유용합니다.
  • 리스트 보기 (List View): 항목들을 간결한 목록 형태로 보여줍니다. 제목과 몇 가지 주요 속성만 표시하여 빠르게 내용을 훑어볼 때 적합합니다.
  • 캘린더 보기 (Calendar View): 날짜 속성이 있는 항목들을 캘린더 형태로 보여줍니다. 일정 관리, 이벤트 계획 등에 활용됩니다.
  • 타임라인 보기 (Timeline View): 시작 날짜와 종료 날짜 속성이 있는 항목들을 간트 차트와 유사한 형태로 보여줍니다. 프로젝트 일정 관리, 작업 기간 시각화에 유용합니다.

실습 과제

9강에서 만든 '나의 영화 감상 목록' 데이터베이스를 사용하여 각 유형의 새 보기를 생성합니다.

  • 새로운 '상태' 속성(볼 예정, 보는 중, 다 봄)을 추가하고, 이 속성을 기준으로 그룹화된 보드 보기를 만듭니다.
  • 영화 포스터를 표시하기 위한 갤러리 보기를 만듭니다. (이를 위해 '이미지' 또는 '파일 및 미디어' 속성을 추가해야 합니다.)
  • '시청일' 속성을 추가하고, 이 속성을 기반으로 한 캘린더 보기를 만듭니다.
