1. 북마크 기능 이해하기

북마크의 진화

Obsidian의 북마크 기능은 기존의 Starred 플러그인을 대체하며 더 강력한 기능을 제공합니다.

2. 북마크 생성과 관리

북마크 추가 방법

방법 1: 컨텍스트 메뉴

  1. 파일/폴더 우클릭
  2. "Bookmark" 선택
  3. 그룹 선택 (선택사항)

방법 2: 명령 팔레트

  1. Ctrl/Cmd + P
  2. "Bookmark" 검색
  3. 원하는 항목 북마크

방법 3: 단축키

커스텀 단축키 설정

Ctrl/Cmd + Shift + B

고급 북마크 기능

검색 쿼리 북마크

// 자주 사용하는 검색 쿼리
tag:#important -tag:#completed
path:Projects file:2024
"meeting notes" file:.md

복잡한 검색 쿼리를 저장하여 빠르게 재사용

그래프 뷰 북마크

특정 필터가 적용된 그래프 뷰 상태를 저장

  • 특정 태그만 표시
  • 날짜 범위 필터
  • 색상 그룹 설정

# 📌 Dynamic Bookmarks Dashboard

## 최근 수정된 북마크 파일
```dataview
TABLE file.mtime as "Modified"
FROM ""
WHERE contains(file.bookmarks, this.file.name)
SORT file.mtime DESC
LIMIT 10
```

## 자주 사용하는 북마크 (수동 추적)
```dataview
TABLE access-count as "Access Count", last-accessed as "Last Used"
FROM #bookmark
WHERE access-count > 10
SORT access-count DESC
```

# Smart Bookmark Collections

## 이번 주 작업 (자동 업데이트)
- 🔍 `file:{{date:YYYY-[W]ww}}` (검색 북마크)

## 현재 프로젝트 (수동 업데이트)
- [[Project Alpha]] - 메인 문서

## 학습 자료 (카테고리별)

- 🔍 `tag:#course file:2024` - 올해 코스

북마크 자동화

Templater로 북마크 관리

<%*
// 프로젝트 시작시 자동 북마크 생성
const projectName = await tp.system.prompt("Project Name:");
const projectPath = `Projects/${projectName}`;

// 프로젝트 구조 생성
await tp.file.create_new("", `${projectName} Dashboard`, false, projectPath);
await tp.file.create_new("", `${projectName} Tasks`, false, projectPath);
await tp.file.create_new("", `${projectName} Notes`, false, projectPath);

// 북마크 그룹 생성 알림
new Notice(`Remember to bookmark the ${projectName} folder!`);

// 프로젝트 대시보드 템플릿
-%>
# 🚀 <%= projectName %> Dashboard

## Quick Links (Bookmark These!)
- [[<%= projectName %> Tasks]]
- [[<%= projectName %> Notes]]
- [[<%= projectName %> Resources]]

## Search Queries to Bookmark
- 🔍 `path:"<%= projectPath %>" tag:active`
- 🔍 `path:"<%= projectPath %>" tag:important`
- 🔍 `"<%= projectName %>" -path:"<%= projectPath %>"` (외부 참조)

6. 모범 사례와 팁

북마크 관리 원칙

정기적인 정리

  • 주간 북마크 리뷰
  • 사용하지 않는 항목 제거
  • 그룹 재구성

일관된 명명

  • 명확한 이름 사용
  • 이모지로 시각적 구분
  • 컨텍스트 포함

계층적 구성

  • 최대 3단계 깊이
  • 논리적 그룹화
  • 자주 사용하는 것 상단

동적 업데이트

  • 프로젝트 진행에 따른 조정
  • 새로운 워크플로우 반영
  • 피드백 기반 개선

생산성 향상 팁

💡 단축키 설정

자주 사용하는 북마크에 개별 단축키 할당

💡 시작 페이지

가장 중요한 대시보드를 북마크하고 시작 페이지로 설정

💡 검색 조합

복잡한 검색 쿼리를 북마크로 저장하여 재사용

💡 임시 북마크

"Temp" 그룹을 만들어 단기 작업용 북마크 관리

실습 프로젝트

🎯 개인화된 북마크 시스템 구축

자신의 워크플로우에 최적화된 북마크 시스템을 설계하고 구현해보세요.

프로젝트 단계

  1. 현재 워크플로우 분석
    • 자주 접근하는 파일/폴더 목록 작성
    • 반복적인 검색 패턴 파악
    • 작업 컨텍스트 분류
  2. 북마크 구조 설계
    • 그룹 계층 구조 결정
    • 명명 규칙 확립
    • 우선순위 정렬
  3. 구현 및 테스트
    • 북마크 생성 및 조직화
    • 1주일 사용 테스트
    • 사용 패턴 추적
  4. 최적화
    • 사용 빈도 기반 재배치
    • 불필요한 북마크 제거
    • 새로운 요구사항 반영

최종 결과물

  • 개인화된 북마크 구조
  • 북마크 관리 가이드라인 문서
  • 자주 사용하는 검색 쿼리 모음
  • 워크플로우별 북마크 세트
