1. 아웃라이너의 개념

구조화된 사고의 도구

아웃라이너는 정보를 계층적으로 구성하여 복잡한 아이디어를 체계적으로 정리하는 강력한 도구입니다.

계층적 구조

주제와 하위 주제의 명확한 관계 표현

접기/펼치기

필요한 정보만 선택적으로 표시

유연한 재구성

드래그로 쉽게 구조 변경

줌 기능

특정 섹션에 집중하여 작업

기본 문법과 조작

- 최상위 항목
  - 하위 항목 1
    - 세부 항목 1.1
    - 세부 항목 1.2
      - 더 깊은 수준
  - 하위 항목 2
    - 세부 항목 2.1
- 다른 최상위 항목

또는 번호 목록:
1. 첫 번째 주제
   1. 하위 주제 A
   2. 하위 주제 B
      1. 세부 내용
      2. 추가 세부 내용

필수 단축키

동작 단축키
들여쓰기 증가 Tab
들여쓰기 감소 Shift + Tab
접기/펼치기 Ctrl/Cmd + .
모두 접기 Ctrl/Cmd + Shift + .
줌 인 Ctrl/Cmd + Click

2. Outliner 플러그인 활용

강화된 아웃라이너 기능

🔄 향상된 이동 기능

  • 항목 위/아래 이동: Ctrl/Cmd + Shift + ↑/↓
  • 전체 브랜치 이동: 하위 항목 포함 이동
  • 드래그 앤 드롭: 마우스로 구조 재배치

/* 번호 목록 스타일 */

3. 효과적인 계층 구조 설계

구조화 원칙

🎯 MECE 원칙

Mutually Exclusive, Collectively Exhaustive

  • 상호 배타적: 중복 없음
  • 전체 포괄적: 누락 없음
- 마케팅 전략
  - 온라인 마케팅
    - 소셜 미디어
    - 검색 엔진
    - 이메일
  - 오프라인 마케팅
    - 인쇄물
    - 이벤트
    - 방송

📏 깊이 관리

적절한 계층 깊이 유지 (보통 3-5단계)

✅ 적절한 깊이
- 주제
  - 대분류
    - 중분류
      - 세부사항
❌ 과도한 깊이
- 주제
  - 분류1
    - 분류2
      - 분류3
        - 분류4
          - 분류5
            - 내용

🔢 일관된 분류

동일 레벨에서 일관된 분류 기준 적용

- 프로젝트 문서
  - 기능별 분류
    - 인증 시스템
    - 결제 시스템
    - 알림 시스템
  - 단계별 분류
    - 기획 문서
    - 개발 문서
    - 운영 문서

6. 실습 프로젝트

🏗️ 지식 체계 구축하기

아웃라이너를 활용해 자신만의 체계적인 지식 관리 시스템을 만들어보세요.

Phase 1: 분야 설정

  1. 관심 분야 3-5개 선정
  2. 각 분야의 주요 주제 도출
  3. 최상위 구조 설계

Phase 2: 계층 구조화

  1. MECE 원칙 적용
  2. 3-4단계 깊이로 구성
  3. 교차 참조 링크 추가

Phase 3: 콘텐츠 채우기

  1. 각 노드에 핵심 정보 추가
  2. 관련 노트 링크
  3. 참고 자료 연결

Phase 4: 동적 요소 추가

  1. Dataview 쿼리 통합
  2. 작업 추적 시스템
  3. 진행 상황 시각화

평가 기준

  • 구조의 논리성과 일관성
  • 정보의 접근성과 탐색 용이성
  • 확장 가능성과 유지보수성
  • 시각적 명확성
